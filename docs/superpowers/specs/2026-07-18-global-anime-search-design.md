# 全部番剧查询页设计

## 目标

提供独立的“查询番剧”页面，按中文名或日文名检索应用已收录的全部 `allAnime` 目录。查询结果使用现有“追番统计”的番剧卡样式，并且每部作品独占一行。

## 根因与修正

此前的查询把匹配后的目录传入周历排期计算，因而改变了当前播出表和我的番剧日历的内容。这与“在全部番剧中查找作品”的意图不符。

新页面直接从 `allAnime` 过滤结果，只用于渲染搜索结果；周历、网络放送列表、季度选择和“我的番剧”继续使用各自原有的数据流，不读取查询词。

## 方案

- 扩展现有页面 URL 状态，使用 `?page=search` 表示查询页，并在左侧导航增加“查询番剧”。浏览器前进、后退和刷新继续复用既有 URL 同步逻辑。
- 将搜索输入框移到查询页。输入为空时提示用户输入中日标题；输入后实时过滤 `allAnime`，匹配规则继续忽略连续空格、英文字母大小写和全角／半角差异。
- 搜索结果复用现有详情按钮与“追番统计”卡片内容：封面、中文名、日文名和所属季度。搜索页结果列表始终为单列，一部番剧一行；点击卡片仍打开现有原生详情弹窗。
- 无匹配时在查询页显示包含查询词的明确空状态。搜索结果不依赖登录、追番列表或已看集数，也不新增 API、数据库字段或番剧数据。

## 界面与无障碍

- 输入使用可见标签、`type="search"` 和中文占位提示。
- 搜索结果卡沿用现有按钮语义、详情弹窗和焦点恢复行为。
- 空结果提示使用 `aria-live="polite"`，让输入后的结果变化可被辅助技术获知。
- 移动端保持单列，无额外横向滚动。

## 验证

- 保留标题匹配模块的单元测试，并补充“目录范围始终为 `allAnime`”与空／无结果状态的页面回归断言。
- 断言周历和网络放送列表仍由未筛选的日历目录驱动，防止查询重新影响当前页面。
- 断言查询页面 URL 状态、导航、结果单列样式和详情按钮连接存在。
- 完成后运行 `npm run lint -- --ignore-pattern .worktrees` 和 `npm test`。

## 非目标

- 不接入站点外部数据库或搜索 API；搜索范围是本应用已收录的全部番剧目录。
- 不按简介、来源链接、播出日期或追番状态检索。
- 不改变季度、周导航、个人追番、逐集已看、认证或数据库语义。
