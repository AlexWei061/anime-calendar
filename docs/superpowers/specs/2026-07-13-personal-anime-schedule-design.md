# 个人番剧时间表设计

## 目标

在现有夏番时间表左侧加入两项页面导航：保留完整季表的“全部夏番”，并新增可选择作品、跨设备同步的“我的番剧”。当前站点继续只允许所有者的 ChatGPT 账号访问；不新增账号、密码或应用自建登录页。

## 页面与交互

- 桌面端的主内容外包一层站点框架。左侧固定页面栏显示“全部夏番”与“我的番剧”。
- “我的番剧”项显示当前已选数量，符合已确认的 B 方案；移动端改为内容顶部的两个页签，不压缩现有日历。
- “全部夏番”保留当前页面的标题、周切换、完整时间轴、网络放送区和详情弹窗，行为不变。
- “我的番剧”页顶部列出本季度全部 66 部作品的复选项。点击任一项立即保存该用户的选择，并更新左栏数量。
- 选择区下方复用同一套周切换、时间轴、网络放送区和详情弹窗，只传入已选作品。未选作品不会出现在任何个人时间表区域。
- 没有选择时显示简短的空状态，不渲染空的时间轴或网络放送区。

## 同步与身份

- 站点继续使用 Sites 现有的 ChatGPT 访问控制；当前访问策略只允许所有者。
- 服务端从平台转发的已认证用户邮箱识别数据归属，绝不接受浏览器提交的用户标识。
- 新增一个只供已认证用户调用的 API：读取该用户已选作品 ID，并以完整 ID 列表覆盖保存选择。请求未带平台身份时返回未认证错误。
- 浏览器首次打开“我的番剧”页时读取同步选择；每次勾选变更后写回，失败时保留原选择并显示可理解的保存失败提示。
- 未来若改为公开访问，只需在“我的番剧”页增加现有 ChatGPT 登录入口；本次不开放站点，也不增加自建登录 UI。

## 数据

- 为 D1 增加一张用户选择表，主键为“用户邮箱 + 番剧 ID”。表中不复制番剧标题、播出信息或封面；这些数据始终来自现有 `data/anime.js`。
- Sites 配置声明一个逻辑 D1 绑定，数据库迁移随部署产物一起发布。
- 保存时先验证每个 ID 都属于当前季的番剧数据，再以一组删除和插入操作使数据库状态与浏览器提交列表一致。

## 边界与可访问性

- 本次不增加搜索、分类、排序、导入导出、共享列表或多套收藏夹。
- 页面切换使用真实按钮并暴露当前页状态；每个番剧选择项可由键盘勾选，并使用完整中文标题作为标签。
- 原有详情弹窗的焦点返回、事件卡片的无障碍标签和窄屏单日议程继续适用于两页。

## 验证

- 先写并运行失败测试，覆盖用户选择 API 的身份要求、未知番剧 ID 拒绝、读写同步和 D1 表定义。
- 覆盖服务端渲染中存在两项页面导航、“我的番剧”选择控件、已选数量和空状态；同时确保完整季表的时间轴、详情弹窗和网络放送功能仍存在。
- 最后运行完整测试、lint、生产构建及空白差异检查；部署时应用 D1 迁移并验证同一 ChatGPT 账号在两台设备读取到同一选择。
