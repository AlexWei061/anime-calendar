# 滴答清单式时间轴周历设计

## 目标

把夏番周表改为滴答清单式连续时间画布：番剧卡片按北京时间直接落在对应时刻；整周在一个页面宽度内显示，不出现横向滚动条。

## 已确认的体验规则

- 桌面周视图始终展示周一至周日和一个共享的左侧时间轴。
- 时间轴是连续画布，不是“按本周出现的时间分组”的列表。节目根据自己的开始时间定位。
- 画布的时间轴标签覆盖北京时间 15:00 至次日 04:00；次日 04:00 后另保留 40px（约 25 分钟）末端缓冲，因此画布总高为 1288px，避免 28:00 开始的卡片被裁切。
- 每小时为 96px，每张节目卡片固定为 40px 高，代表约 25 分钟。
- 原始 YUC 时间达到 24:00 时，左侧轴显示为“次日 HH:MM”，节目仍放在原始放送日期列。
- 同一天、同一时刻的多部节目在同一垂直位置横向分栏，不允许上下堆叠。
- 相隔不足 25 分钟而会相互遮挡的节目也使用并行分栏，保持每张卡片可点击。
- 卡片固定为“左封面、右文字”：封面保持完整 3:4 比例且从不因空间不足被隐藏；中文番名只显示一行，可用省略号截断；集数在空间允许时显示在第二行。
- 周视图不设 `min-width`，不使用内部横向滚动。七个日列平分可用页面宽度。
- 窄屏保留现有单日议程入口，而不是压缩七天列或引入横向滚动。

## 组件与数据流

1. `eventsForWeek` 继续生成带原始 `time`、日期、集数和本地封面 URL 的节目事件。
2. 日历辅助函数将时间转为距 15:00 的分钟偏移，并为 25 分钟视觉区块计算重叠分栏。
3. 页面渲染一个共享时间轴、七个固定日期列和每列的绝对定位节目按钮。每个按钮通过 CSS 自定义属性取得开始位置、分栏索引和分栏总数。
4. CSS 以每分钟 1.6px 定位卡片；卡片高度固定 40px；同一分栏组按等宽列展示。
5. 当前的详情弹窗、周切换、北京时间、YUC 数据、网络放送区与本地封面资产保持不变。

## 响应式与可访问性

- 桌面周视图只纵向滚动；日期标题和时间轴沿用现有可读色彩与焦点样式。
- 图片保留空 `alt`，按钮的完整中文和日文标题、日期、显示时间与集数继续放在 `aria-label` 中。
- 移动端继续使用单日议程，避免七天列造成横向滚动或把封面缩到不可辨认。

## 验证

- 添加日历辅助函数测试：15:00、24:00 和 25:00 的位置及显示时间；25 分钟重叠的分栏；同刻节目共享垂直起点。
- 更新服务端渲染测试：渲染连续时间轴、七日时间画布、定位变量、左侧封面和同刻横向分栏。
- 断言旧的逐日 `.time-groups` 布局与横向滚动最小宽度不再是周视图实现。
- 运行 `npm test` 和 `git diff --check`。

## 不在范围内

- 不修改 YUC 数据、集数策略、翻译、封面下载或定期更新机制。
- 不增加拖拽、个人待办、账号、提醒或日历写入功能。
