# 每周时间轴裁剪设计

## 目标

让周视图只显示该周有定时番剧的最小时间范围，移除上下无节目时段造成的大面积空白。

## 计算规则

每次周视图的节目数据确定后，按节目当前的视觉放送时间计算边界：

- 最早节目向下取整到整点，作为时间轴开始。
- 最晚节目加上现有卡片时长后向上取整到整点，作为时间轴结束，确保卡片完整显示。
- 计算结果限制在当前季度原有允许范围内。
- 已按前一日“次日 00:00–04:59”展示的凌晨节目，使用其现有视觉时间参与计算，不改变日期归属或详情中的真实播出时间。
- 某周没有任何定时节目时，使用该季度原本的完整时间范围，避免渲染零高度的日历。

例如，一周最早和最晚节目都是 16:00，则视图显示 16:00–17:00；最晚节目为 23:40，则显示到能容纳卡片的下一个整点。

## 实现边界

在日历工具模块增加纯函数，输入本周事件、季度默认范围与卡片时长，输出时间轴开始和结束分钟数。页面以这个结果生成刻度、容器高度和节目偏移。

手机端按日节目列表、未定时节目列表、周切换、季节切换和节目详情不改变。

## 验证

- 测试普通周的整点裁剪和卡片结束取整。
- 测试跨日视觉时间参与裁剪。
- 测试无定时节目周回退到默认范围。
- 运行全量测试、lint 和生产构建。
