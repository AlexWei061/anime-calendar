# 已看集数标记设计

## 目标

登录同一 ChatGPT 账号的用户可以为日历中的一次周更标记“已看”。标记在桌面时间轴和移动端单日议程中即时显示为右上角勾选与轻微标灰，并在刷新或换设备后保留。

## 页面与交互

- 每个有固定周播时刻的日历集数卡都分为两个独立控件：主体按钮打开现有详情弹窗，右上角小方块切换该次放送的已看状态。
- 未看时小方块为空；已看时显示勾号，卡片、标题和封面使用低对比度灰态，但集数仍清晰可读。
- 小方块有明确的中文无障碍名称，使用 `aria-pressed` 表示状态，并具有足够的键盘焦点与触控区域。
- 同一事件组件同时被桌面时间轴和移动端日程使用，因此两种视图的状态与行为一致。
- 没有固定时刻的“网络放送／固定时刻未列出”卡片不提供已看标记；它们不代表可定位到某一周的一次集数更新。

## 同步与身份

- 新增一个只供已认证用户调用的 API。服务端继续调用 `getChatGPTUser()` 获取邮箱；浏览器不能提交或决定用户身份。
- 页面首次加载读取该账号所有已看更新；读取期间小方块不可操作，读取失败或未登录时不乐观改变状态并显示简洁错误提示。
- 点击小方块时，页面先切换显示状态，再请求保存；请求失败则恢复原状态并提示重试。
- 单次写入只改变被点击的更新，不覆盖该用户其他已看记录，避免多设备或多标签页的无关标记被覆盖。

## 数据与校验

- D1 新增 `anime_episode_views` 表，复合主键为 `user_email`、`anime_id`、`episode_start`、`episode`。
- 一条记录代表一次日历更新；通常 `episode_start` 与 `episode` 相同，首播连播多集时保留完整的集数范围。因此用户点击“第 1—3 集”会把这一整次更新标为已看。
- 不复制标题、时间、封面等目录资料。页面始终从现有番剧目录与 `eventsForWeek` 计算展示内容。
- 写入时验证番剧 ID 必须存在，并验证集数范围为正整数、起始集不大于结束集、结束集不超过该番剧总集数。数据库主键防止重复记录。
- Drizzle schema 和生成的迁移必须一起更新；保留现有逻辑 D1 绑定名 `DB`。

## 实现边界

- 不改变个人追番列表、季度选择、周导航、详情弹窗、网络放送资料或 Sites 登录路由。
- 不新增自建账号、权限模型、进度百分比、自动标记、批量标记、搜索或统计。
- 为避免无效的交互式元素嵌套，现有整张 `<button>` 日历卡改为非交互容器，内部包含一个详情按钮和一个独立的已看按钮；原有详情按钮语义和关闭后的焦点返回继续保留。

## 验证

- 先写失败测试：已看更新数据的格式、番剧白名单和集数范围校验；路由必须验证 ChatGPT 身份并使用新表。
- 覆盖构建后的 HTML：桌面时间轴和移动端都存在已看控制，控制具有可访问的状态语义，详情按钮仍有 `aria-haspopup="dialog"`，且没有嵌套按钮。
- 覆盖页面源码契约：已看状态在首次加载时读取，单项切换失败会回滚，桌面与移动复用同一事件渲染逻辑。
- 完成后运行 `npm run lint -- --ignore-pattern .worktrees`、`npm test` 和 `git diff --check`。
