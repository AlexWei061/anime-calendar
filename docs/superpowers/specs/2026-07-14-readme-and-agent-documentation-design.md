# README 与开发约定同步设计

## 目标

把仍是 vinext 默认模板的 `README.md` 改为面向本项目使用者的说明，并把
`AGENTS.md` 中已经过期的项目事实同步为当前实现。两份文档分别服务于使用者与
开发者，避免对同一数据来源、时间轴和验证命令给出相互矛盾的描述。

## 范围

### README

- 说明这是 2026 年 1 月、4 月与 7 月番的中文播出日历。
- 说明桌面周历、移动日程、跨季度翻周、个人追番和本地封面的现有功能。
- 明确资料口径：YUC 提供中文名与封面；1、4 月历史排期使用 AniList 的首播、
  周播时间与集数；7 月番使用 YUC 排期。
- 保留最少的运行、验证与生成数据命令，并注明含 `.worktrees/` 时的 lint 命令。

### AGENTS.md

- 更新项目定位、数据模块地图和封面目录约定。
- 说明历史资料由生成脚本写入 `data/yuc-history-2026.js`，不手工编辑生成结果。
- 将桌面时间轴规则更新为按当前周事件动态裁切，空周回退到 05:00 至次日 05:00。
- 说明季度选择器仅负责定位到季度首周；周历使用全年已收录资料连续前后翻周。
- 将“全部夏番”更新为“播出表”，并修正 lint 命令。

## 非目标

- 不修改数据、前端、数据库或部署配置；仅同步直接断言 README 内容的既有测试。
- 不扩大为 2020 年至今的完整资料导入。
- 不保留与项目无关的 vinext、D1 示例或通用鉴权教程。

## 验收

- README 不再称自己为 vinext starter，且所有命令与 `package.json` 一致。
- AGENTS 的资料来源、封面路径、跨季度与时间轴规则能对应当前源码。
- 文档检查不产生空白、冲突或失效链接；不改动运行时代码。
