# 番剧查询设计

## 目标

让用户在“播出表”和“我的番剧”中按中文名或日文名实时查询，并使桌面周历、移动议程和网络放送列表始终显示同一组匹配结果。

## 范围

- 在日历标题下方提供一个可清空的原生搜索输入框；“追番统计”页不显示该输入框。
- 查询只在当前页面的既有番剧范围内工作：播出表使用全部目录，我的番剧使用当前用户已收藏的目录。
- 匹配 `titleZh` 和 `titleJa`，忽略首尾空格、连续空格、英文字母大小写及全角／半角差异。
- 空查询恢复原有显示；没有匹配时显示明确的空状态。
- 匹配到的作品若本周没有排期，维持既有“当天没有排定放送”的日历语义，不误报为未找到作品。

## 方案

新增一个仅负责规范化和比较标题的纯 JavaScript 辅助模块。页面保留查询输入状态，将当前页面的番剧目录交给该模块过滤，再将过滤结果传给现有的 `eventsForWeek`、`dateOnlyEventsForWeek` 与网络放送列表。

这样时间轴边界、桌面卡片和移动议程继续共享现有排期计算，不新增 API、数据库字段、路由参数或数据文件改动。收藏、已看、季度选择与周切换保持原行为。

## 界面与无障碍

- 输入使用 `type="search"`、可见标签和中文占位提示；浏览器原生清除按钮可恢复结果。
- 搜索栏使用现有标题控制区的视觉语言。小于等于 860px 时占满可用宽度，避免与季度选择器挤在同一行。
- 无匹配提示使用 `aria-live="polite"`，并包含用户输入的查询词。

## 验证

- 为标题匹配模块添加单元测试，覆盖中文名、日文名、大小写／全半角／空白规范化及空查询。
- 为页面与样式的静态回归测试补充搜索输入、共享过滤数据流、空状态和移动端样式断言。
- 实现完成后运行 `npm run lint -- --ignore-pattern .worktrees` 与 `npm test`。

## 非目标

- 不提供跨季度搜索结果页或自动跳转。
- 不搜索统计页、来源链接、简介或站点外部数据。
- 不修改番剧目录、认证、追番存储或已看记录。
