# 搜索结果追番进度设计

## 目标

让“查询番剧”页面的每个结果显示当前用户是否已加入追番，以及实际已看集数，沿用“追番统计”现有的状态徽标、进度条和详情弹窗。

## 当前问题

搜索页已经从全部 `allAnime` 目录生成单列结果，但只显示所属季度。追番列表只在“我的番剧”和“追番统计”页面加载，因此搜索页无法判断作品是否在追，也没有已看集数可展示。

## 方案

- 用户进入搜索页时，与“我的番剧”和“追番统计”一样读取当前 ChatGPT 账号的追番列表；已看记录继续使用现有读取逻辑和数据。
- 对当前搜索结果调用既有 `progressForAnime`，按番剧 ID 建立进度查找。每张搜索卡继续调用 `statisticsAnimeCard`，不新增卡片、API 或数据库结构。
- 已加入追番的作品使用既有状态：`未开始`、`在追` 或 `已看完`，并显示“已看 X / N 集”和进度条。
- 未加入追番的作品显示“未追番”，但仍显示其历史已看集数和对应进度条；因此取消追番后保留的已看记录不会被隐藏或误报为零。
- 读取期间显示“正在读取追番进度…”。若追番或已看记录读取失败，仍显示搜索结果本身，但显示“进度暂不可用”，不把无法读取误标为“未追番”或“已看 0 集”。

## 边界

- 搜索范围仍固定为 `allAnime`，不按追番状态过滤，也不影响周历、移动议程、网络放送或季度选择。
- 追番状态和已看集数仍以服务器端按 ChatGPT 邮箱隔离的数据为准；浏览器不提供身份，也不新增写入行为。
- 不改动详情弹窗、选番控件、逐集已看切换、D1 schema、迁移或 API 路由。

## 无障碍与加载

- 卡片沿用现有按钮语义、`aria-haspopup="dialog"` 与关闭后的焦点恢复。
- 进度读取状态和读取失败提示使用现有 `aria-live="polite"` 语义。
- 搜索结果始终保持当前的一部一行布局，移动端不增加额外交互。

## 验证

- 在 `tests/rendered-html.test.mjs` 增加回归断言：搜索页会加载追番列表、从搜索结果派生既有进度、按是否收藏显示正确状态，并在数据不可用时避免伪造进度。
- 保留现有断言：搜索始终使用 `allAnime`，周历和网络列表不使用搜索结果。
- 完成后运行 `npm run lint -- --ignore-pattern .worktrees` 与 `npm test`。

## 非目标

- 不按进度、已看状态或追番状态筛选或排序搜索结果。
- 不在搜索页新增收藏、取消收藏或标记已看的写入控件。
- 不修改未登录时的认证流程或站点访问权限。
