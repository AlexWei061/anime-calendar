# 我的番剧地址状态与选择区折叠设计

## 目标

让“我的番剧”在刷新后保持当前页面，并减少日常查看日程时选择列表占用的空间。

## 页面状态

- 当前页由地址参数 `page` 表示：`?page=mine` 打开“我的番剧”；缺少或无效参数时打开“全部夏番”。
- 点击侧栏或移动端页签时同步更新地址参数；浏览器前进、后退时按地址恢复页面。
- 服务端首屏继续渲染默认“全部夏番”，客户端加载后读取地址参数，避免服务端与客户端首屏不一致。

## 选择区

- “我的番剧”页面中的“本季度想追什么？”默认折叠。
- 使用原生 `details` / `summary`，不增加 React 折叠状态或新依赖。
- 展开后保留现有的复选列表、同步提示、读取和保存错误提示及禁用逻辑。

## 验证

- 先添加失败的源码行为断言，要求页面读取并写入 `page=mine`，并监听浏览器历史变化。
- 要求页面使用无 `open` 属性的原生折叠组件及可见的选择区标题。
- 运行完整测试、lint 和生产构建。
