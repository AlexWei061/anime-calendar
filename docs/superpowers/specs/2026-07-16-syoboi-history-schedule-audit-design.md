# しょぼいカレンダー历史排期校正设计

## 目标

以 しょぼいカレンダー已发生的日本电视播出记录，校正当前站点从 2020 年起已收录作品的首播日期、北京时间、每周固定时刻、首播连播和实际改档。站点继续展示北京时间；不改变 YUC 的中文名、封面、季度归属或稳定作品 ID。

## 来源与取值口径

- 通过 しょぼいカレンダー公开 `db.php` 的 `TitleLookup`、`ProgLookup` 与频道资料读取数据；请求使用专属 User-Agent，并按批次顺序执行。
- 只采用日本电视台的非重播记录。对同一作品，首播取最早的有效电视播出；该频道成为作品的排期频道。
- 将日本时间（UTC+9）严格换算为北京时间（UTC+8），包括源站使用 `24:00` 以上表示的深夜时段。
- 2020 至 2025 年全部纳入。2026 年只覆盖在校正运行时已经实际播出的集数；未来排期仍沿用现有 YUC／官方资料，避免把社区预测当作事实。
- しょぼい记录不足、标题无法唯一对应、频道仅有重播或无法得出稳定规则时，保留当前资料并生成待复核报告，绝不猜测。

## 数据与匹配

### 原始快照

新增按年生成的 しょぼい资料快照。每个快照保留：站内 TID、站名、候选日文标题、采用频道、首播和逐集实际播出时间、获取日期和来源链接。快照仅保存本项目已收录作品需要的字段，不镜像完整站点数据。

### 标题解析

1. 以现有作品的 `titleJa`、AniList 原始日文标题和已维护的别名表构造规范化标题索引。
2. 再用季度及首播年份约束候选范围；只有一对一匹配才自动采用。
3. 同名续作、分割放送、重制版和匹配多个 TID 的作品全部写入报告，等待显式别名；不以标题相似度自动猜测。

### 日历字段

- 保留现有 `premiereDateBeijing`、`scheduleWeekday`、`beijingTime` 和 `premiereEpisodeCount` 作为展示摘要，但改由已选频道的实际记录生成。
- 新增可选的逐集 `episodeSchedules`。每项含集数范围、北京时间日期和时刻，用于停播、改档、首播多集和分割放送；没有例外的连续周播不重复存储。
- 记录 `scheduleSourceName`、`scheduleSourceUrl` 与 `scheduleChannel`，使排期来源和取值频道可审计，而原有 YUC `sourceUrl` 继续指向目录页。
- `eventsForWeek` 优先使用逐集记录；没有逐集例外时沿用当前“首播 + 固定周播”展开逻辑。超过已验证逐集范围的 2026 节目不把历史记录外推为已发生事实。

## 导入与更新流程

1. `generate-syoboi-history` 获取标题、频道和逐集播出资料，写入对应年份的本地生成快照，并输出匹配／未匹配／歧义统计。
2. `generate-yuc-history-pilot <year>` 读取现有 YUC 目录、AniList 集数资料和同年的 しょぼい快照，优先合并已验证的 しょぼい排期。
3. 仅在排期字段变化时改写 `data/yuc-history-<year>.js`；封面路径和图集映射不变，因此不触发封面重新下载或图集重建。
4. 先以 2020 年做端到端试点，再按年执行 2021 至 2025，最后处理 2026 的已播记录。每批生成后检查差异和待复核报告，避免一次不可审计的大型改动。

## 例外处理

- 首播同时播出多集时，写入连续集数范围并保留 `premiereEpisodeCount`。
- 停播、临时改时、总集篇、分割放送和复播不改写为虚假的固定周播；有实际集数记录时以 `episodeSchedules` 覆盖。
- 网络放送、没有固定电视台时刻或仅有无法验证的资料继续保持 `null`，在页面的“网络放送／固定时刻未列出”区域显示。
- 不变更作品 ID、YUC 标题、封面、集数和季度归属；这些不属于本次排期校正。

## 验收与测试

- 为标题唯一匹配、歧义拒绝、东京时间换算、`24:00` 以上时段、最早非重播频道选择、首播连播、停播和改档编写纯数据回归测试。
- 对每个已更新年份验证：季度数量不变、作品 ID 唯一、`catalogCount` 相等、封面图集仍可解析，以及每项改写排期都有来源链接。
- 用 ID:INVADED、进击的巨人最终季前半与 Part 2 作为跨年首播、分割放送和固定周播的回归样例。
- 每个批次运行 `npm run lint -- --ignore-pattern .worktrees` 与 `npm test`；所有年份完成后，再构建、提交并发布同一个私有 Sites 版本。

## 非目标

- 不导入 SP、剧场版、纯网络首发或重播目录。
- 不把 しょぼい的全部频道表加入运行时包。
- 不对未唯一匹配的作品使用模糊匹配或人工猜测。
- 不把未播出的 2026 节目当作历史实际播出记录。
