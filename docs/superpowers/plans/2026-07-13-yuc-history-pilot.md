# YUC 历史试点 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 2026 年 1 月、4 月试点改为 YUC 名称与封面，并把 AniList 的凌晨排期显示为前一天的“次日”时段。

**Architecture:** 新生成器以 YUC 月表的 180px 资料卡为目录，输出本地静态季表；它将中文名、日文名、HTTPS 封面和 YUC 页面来源写入记录，再按规范化日文标题关联现有 AniList 静态排期。日历层将凌晨播放时间转换成仅供布局使用的跨日日期和 24:00–28:59 时刻，同时保留实际播放日期和时刻给详情弹窗使用。

**Tech Stack:** Next.js/Vinext、React、Node.js 内置 fetch、Node test runner、ESLint。

- [ ] **Step 3: 实现跨日布局转换**

```js
function visualSchedule(date, time) {
  const minutes = timeToMinutes(time);
  if (minutes >= 5 * 60) return { date, time, broadcastDate: date, broadcastTime: time };

  const visualMinutes = minutes + 24 * 60;
  return {
    date: addDays(date, -1),
    time: `${String(Math.floor(visualMinutes / 60)).padStart(2, "0")}:${String(visualMinutes % 60).padStart(2, "0")}`,
    broadcastDate: date,
    broadcastTime: time,
  };
}
```

In `eventsForWeek`, build the logical episode date first, pass it through `visualSchedule`, then filter with the visual date so week edges are correct. In `page.tsx`, select `event.broadcastDate` and `event.broadcastTime` for the dialog’s actual broadcast detail while retaining `event.date` and `event.time` for the grid.
